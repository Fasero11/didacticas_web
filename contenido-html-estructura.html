Sesión 2: Estructura y etiquetas básicas

Todo documento HTML tiene una estructura organizada que permite al navegador leer, interpretar y mostrar correctamente el contenido de una página web.

Esta estructura funciona como el esqueleto de la página: define qué información pertenece a la configuración interna del documento y qué contenido será visible para el usuario.

1. ¿Por qué es importante la estructura HTML?

La estructura HTML es importante porque ayuda a mantener el código ordenado, comprensible y fácil de modificar. Gracias a ella, el navegador puede identificar cada parte del documento y mostrar la página de forma correcta.

Una página bien estructurada también facilita el trabajo con CSS y JavaScript, ya que cada elemento está colocado en el lugar adecuado.

2. Partes principales de un documento HTML

Un documento HTML básico está formado por cuatro partes principales:

  1. <!DOCTYPE html>

    Indica al navegador que el documento utiliza HTML5. Siempre debe aparecer en la primera línea del archivo.

  2. <html>

    Es la etiqueta raíz del documento. Dentro de ella se coloca todo el contenido de la página.

  3. <head>

    Contiene información interna del documento, como el título de la página, configuraciones y enlaces a archivos externos.

  4. <body>

    Contiene todo el contenido visible para el usuario: textos, imágenes, enlaces, vídeos, botones y otros elementos.

3. Ejemplo de estructura básica

El siguiente ejemplo muestra cómo se organizan estas partes dentro de un documento HTML: 

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8">
    <title>Mi primera página</title>
  </head>

  <body>
    <h1>Bienvenidos</h1>
    <p>Esta es mi primera página web.</p>
  </body>
</html>

En este ejemplo, la parte <head> contiene información para el navegador, mientras que la parte <body> contiene el contenido que se muestra en la página.

4. Elementos dentro de <head>

La sección <head> no muestra contenido directamente en la página, pero contiene información necesaria para que el documento funcione correctamente.

  • <title>

    Define el título que aparece en la pestaña del navegador.

  • <meta>

    Incluye configuraciones del documento, como la codificación de caracteres.

  • <link>

    Conecta archivos externos, por ejemplo una hoja de estilos CSS.

  • <script>

    Permite enlazar o escribir código JavaScript.

5. Elementos dentro de <body>

La sección <body> contiene todos los elementos que el usuario puede ver e interactuar dentro de la página web.

  • <h1> - <h6>

    Se utilizan para crear títulos y subtítulos.

  • <p>

    Sirve para escribir párrafos de texto.

  • <img>

    Permite insertar imágenes en la página.

  • <a>

    Crea enlaces hacia otras páginas o sitios web.

  • <ul> / <ol>

    Permiten crear listas no ordenadas y ordenadas.

  • <video>

    Permite insertar vídeos en la página.

En las siguientes sesiones se estudiarán con mayor detalle estas etiquetas, incluyendo texto, enlaces, imágenes, listas, tablas y videos, para comprender mejor su estructura, funciones y uso práctico en el desarrollo web.

6. Etiquetas semánticas de organización en HTML

Además de las etiquetas básicas de estructura como <head> y <body>, HTML también incluye etiquetas semánticas que permiten organizar mejor el contenido de una página web.

Estas etiquetas ayudan a dividir visual y estructuralmente el sitio, facilitando la navegación, la accesibilidad y el mantenimiento del código.

  • <aside>

    Se utiliza para crear contenido lateral o complementario, como barras laterales, menús secundarios, índices o información adicional.

    Ejemplo: barra lateral con sesiones o índice de contenido.

  • <nav>

    Define una sección de navegación que contiene enlaces importantes, como menús principales o secundarios.

    Ejemplo: menú superior con enlaces a Inicio, Contenido o Autoevaluación.

  • <div>

    Es un contenedor genérico utilizado para agrupar elementos y aplicar estilos o estructuras personalizadas.

    Ejemplo: agrupar botones, tarjetas o bloques de contenido.

  • <section>

    Representa una sección temática del contenido, agrupando información relacionada dentro de la página.

    Ejemplo: separar una lección en introducción, teoría y práctica.

El uso correcto de estas etiquetas mejora la organización del documento y hace que la página sea más clara tanto para desarrolladores como para navegadores.
