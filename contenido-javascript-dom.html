Sesión 11: Interacción con el DOM

1. ¿Qué es el DOM?

El DOM (Document Object Model) es la estructura en forma de árbol que el navegador crea a partir de tu HTML. JavaScript utiliza el DOM para "conectarse" a la página, permitiendo cambiar textos, colores o eliminar elementos sin necesidad de escribir código HTML nuevo. Es, básicamente, el puente que convierte un documento estático en una aplicación viva.

2. Seleccionar elementos

Antes de cambiar algo, JavaScript necesita encontrarlo. Los métodos más utilizados para "atrapar" elementos son:

  • getElementById: Busca un elemento por su ID único (ej. document.getElementById("miBoton")).
  • Es el más versátil, ya que usa la misma sintaxis que CSS para buscar por etiqueta, clase o ID (ej. document.querySelector(".clase-texto")).

3. Modificar contenido y estilo

Una vez seleccionado el elemento, podemos transformarlo usando propiedades específicas:

  • innerHTML / textContent: Sirven para cambiar el texto o el HTML que hay dentro de una etiqueta.
  • style: Permite modificar el CSS directamente (ej. elemento.style.color = "red").
  • classList: Se usa para añadir o quitar clases de CSS, lo cual es la mejor forma de cambiar diseños de golpe.

4. Eventos

Los eventos son acciones que ocurren en la página, como un clic, mover el ratón o pulsar una tecla. Usamos addEventListener para decirle al navegador: "Cuando pase esto, haz aquello".

  • Ejemplo: Al hacer click en un botón de "Modo Oscuro", JavaScript cambia el color de fondo de toda la página.

5. Resumen

El DOM es el puente que permite a JavaScript conectarse con el HTML para transformar una página estática en una experiencia dinámica. A través de selectores, el código localiza elementos específicos para modificar sus textos, colores o estructuras de forma inmediata. Gracias a los eventos, la web puede "escuchar" y reaccionar a las acciones del usuario, como clics o movimientos del ratón. En definitiva, la interacción con el DOM es lo que permite que una página web funcione como una aplicación viva y no solo como un documento de lectura.

6. Link y actividad

Para profundizar en el DOM visite:

W3Schools

Tarea cooperativa mediante puzzle de Aronson: En grupos de 4, cada alumno trabaja una microtarea distinta. Primero se reúne con compañeros que tienen el mismo rol para convertirse en experto y después vuelve a su grupo base para explicar su parte y construir el producto final común.

  1. Experto 1 - ¿Qué es el DOM?: Explica el DOM como puente entre HTML y JavaScript. Dibuja o describe una pequeña estructura de árbol con una página sencilla.
  2. Experto 2 - Seleccionar elementos: Selecciona un elemento con getElementById y otro con querySelector. Explica la diferencia entre ambos métodos.
  3. Experto 3 - Modificar contenido y estilo: Cambia el texto de un elemento y añade o quita una clase con classList para modificar su aspecto.
  4. Experto 4 - Eventos: Añade un addEventListener a un botón para que la página reaccione al hacer clic.

Producto final del grupo: una pequeña práctica conjunta donde aparezcan integradas las cuatro aportaciones y cada miembro pueda justificar su decisión.

Respuesta modelo:

<button id="boton-cambiar">Cambiar texto</button>
<p class="mensaje">Texto inicial</p>

<script>
const boton = document.getElementById("boton-cambiar");
const mensaje = document.querySelector(".mensaje");

boton.addEventListener("click", function() {
  mensaje.textContent = "Texto cambiado con JavaScript";
  mensaje.classList.add("destacado");
});
</script>
