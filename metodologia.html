Metodologías a aplicar

Para impartir estos contenidos técnicos de forma dinámica y participativa, se combinarán dos metodologías activas: el aula invertida y el puzzle de Aronson.

Flipped Classroom o Aula Invertida

Supone una inversión del método tradicional. Los alumnos estudiarán de forma autónoma los conceptos teóricos previos a cada clase mediante recursos facilitados por el docente, como vídeos o lecturas.

Esto permite que el tiempo presencial en el aula se aproveche para resolver dudas, realizar prácticas grupales e iniciar debates.

  • Aumenta el interés del alumnado.
  • Favorece el aprendizaje personalizado.
  • Ayuda tanto a alumnos con facilidad como a quienes presentan más dificultades.

Puzzle de Aronson o Jigsaw

Es una técnica de aprendizaje cooperativo basada en roles, ideal para su aplicación en Educación Secundaria.

Su objetivo es mejorar el rendimiento, fomentar la autonomía, madurar el conocimiento y desarrollar habilidades sociales.

El alumnado se divide en grupos base o nodriza de cuatro miembros. Cada integrante se especializará en un subtema, convirtiéndose en experto. Después, se reunirá con los expertos de otros grupos para preparar la documentación y, finalmente, volverá a su grupo base para explicar su parte al resto de compañeros.

  • Promueve la cooperación entre iguales.
  • Reparte la responsabilidad del aprendizaje.
  • Permite que todos los alumnos aprendan el contenido completo.

Desarrollo de las sesiones

Antes de iniciar los módulos, el profesor dedicará un tiempo previo a explicar la metodología de Aronson y a configurar los grupos base de cuatro miembros.

El curso se divide en tres bloques: HTML, JavaScript y CSS. Cada bloque contará con cuatro clases o sesiones, correspondientes a los cuatro temas de cada materia.

  • En casa: los alumnos revisan de forma autónoma el material general del tema del día.
  • En clase, grupo de expertos: se agrupan con compañeros que tienen el mismo rol para profundizar en su subtema.
  • En clase, grupo nodriza: regresan a su grupo original para explicar los conceptos aprendidos.
  • Práctica final: realizan juntos una actividad, documento o ejercicio final relacionado con el tema.

Organización de roles por sesión

En cada sesión se mantendrá la estructura de grupos base de cuatro alumnos, pero los roles de experto cambiarán según el contenido trabajado. De esta forma, cada miembro del grupo tendrá una responsabilidad concreta y necesaria para que el equipo pueda comprender el tema completo.

La dinámica se organizará en tres momentos. Primero, cada alumno revisará en casa el contenido general de la sesión. Después, en el aula, se reunirá con otros alumnos que tengan el mismo rol para profundizar en su microtema. Finalmente, volverá a su grupo base para enseñar su parte y construir una actividad común con las aportaciones de los cuatro expertos.

Roles del módulo HTML

Sesión Experto 1 Experto 2 Experto 3 Experto 4
1. Introducción a HTML Explica qué es HTML y por qué es un lenguaje de marcado, no de programación. Describe para qué sirve HTML y dónde se usa: blogs, tiendas online o páginas informativas. Identifica quién utiliza HTML y qué ventajas tiene aprenderlo. Relaciona HTML con CSS y JavaScript dentro de una web moderna.
2. Estructura básica Explica la importancia de <!DOCTYPE html> y <html>. Analiza el elemento <head> y etiquetas como <title>, <meta> o <link>. Explica el papel del <body> y de etiquetas semánticas como <nav> o <aside>. Describe elementos visibles del body: títulos, párrafos e imágenes.
3. Etiquetas de texto Explica por qué las etiquetas de texto dan significado y mejoran la organización. Diferencia títulos, párrafos y etiquetas visuales o semánticas como <b>, <strong>, <i> y <em>. Muestra cómo organizar información con listas <ul> y <ol>. Explica cómo crear tablas con <table>, <tr>, <th> y <td>.
4. Enlaces e imágenes Explica los enlaces y el atributo href. Trabaja enlaces internos y anclas mediante identificadores id. Explica imágenes con <img>, src y alt. Introduce vídeos en HTML con <video>, controls y src.

Roles del módulo CSS

Sesión Experto 1 Experto 2 Experto 3 Experto 4
1. Introducción a CSS Diferencia una web HTML sin estilos de una web con CSS. Explica cómo modificar colores de texto y fondos. Trabaja tamaños y cajas visuales con propiedades como font-size, bordes o sombras. Explica cómo separar contenido y diseño enlazando styles.css.
2. Selectores CSS Aplica selectores de etiqueta a elementos como todos los párrafos. Usa selectores de clase para reutilizar estilos en varios elementos. Usa selectores de ID para elementos únicos. Explica selectores combinados como nav a.
3. Box Model Explica el concepto de caja y el uso de margin. Trabaja el padding como espacio interior. Explica el border y sus estilos. Analiza box-sizing: border-box para controlar el tamaño real de las cajas.
4. Flexbox y responsive Explica display: flex para organizar elementos. Trabaja la alineación horizontal con justify-content. Trabaja la alineación vertical con align-items. Explica el diseño responsive con flex-wrap.

Roles del módulo JavaScript

Sesión Experto 1 Experto 2 Experto 3 Experto 4
1. Introducción a JavaScript Explica qué es JavaScript y cómo aporta interactividad. Explica cómo integrarlo en HTML con <script>. Usa la consola del navegador y console.log(). Explica comentarios de una línea y de varias líneas.
2. Lógica y sintaxis Diferencia variables y constantes con let y const. Identifica datos simples: strings, números y booleanos. Organiza información con arrays y objetos. Explica decisiones mediante condicionales if.
3. Interacción con el DOM Explica qué es el DOM y su papel como puente entre HTML y JavaScript. Selecciona elementos con getElementById o querySelector. Modifica contenido y estilos con innerHTML, style o classList. Usa eventos con addEventListener.
4. Funciones Explica qué es una función y por qué permite reutilizar código. Describe la estructura de una función con function, nombre y llaves. Diferencia parámetros y argumentos. Explica el valor de retorno con return.
