Sesión 4: Funciones y reutilización

1. ¿Qué es una función?

Una función es un bloque de código reutilizable diseñado para realizar una tarea específica. En lugar de escribir las mismas instrucciones diez veces, las agrupas en una función y simplemente la "llamas" cada vez que la necesites. Es como una receta de cocina: la escribes una vez y puedes usarla siempre que quieras cocinar ese plato.

2. Estructura de una función

Para crear una función básica, necesitamos tres partes fundamentales:

  • La palabra function: Indica al navegador que vamos a declarar una función.
  • Nombre de la función: Un nombre descriptivo (ej. saludar o calcularTotal).
  • El cuerpo { }: Todo el código que está dentro de las llaves es lo que se ejecutará al llamar a la función.

3. Parámetros y argumentos

Las funciones pueden recibir información para trabajar con ella.

  • Parámetros: Son las variables que ponemos entre los paréntesis ( ) al crear la función. Actúan como huecos vacíos.
  • Argumentos: Es el valor real que le enviamos a la función cuando la usamos (ej. enviarle el nombre "Carlos" al parámetro usuario).

4. El valor de retorno (return)

Muchas veces no queremos que la función solo muestre un mensaje, sino que nos devuelva un resultado (como una operación matemática). Para ello usamos la palabra return. Una vez que la función llega al return, se detiene y entrega el valor final a quien la llamó.

5. Resumen

Las funciones son bloques de código reutilizables que ejecutan tareas específicas cuando son invocadas, evitando la duplicación de instrucciones. Pueden recibir información mediante parámetros y devolver resultados procesados a través de la sentencia return. Al definirlas una sola vez, facilitamos el mantenimiento del código y mejoramos la organización lógica de nuestra aplicación. Su uso es esencial para crear programas escalables y fáciles de entender.
