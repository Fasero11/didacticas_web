Sesión 1: Introducción a JavaScript

JavaScript es el lenguaje que permite convertir páginas web estáticas en experiencias interactivas y dinámicas. Mientras que el HTML construye la estructura de la web, JavaScript actúa como el sistema nervioso que le da vida, permitiendo que la página reaccione a lo que el usuario hace en tiempo real.

Hoy en día, es una de las tecnologías más potentes del mundo, utilizada no solo para sitios web, sino también para crear aplicaciones móviles y juegos complejos que funcionan directamente en el navegador.

1. ¿Qué es JavaScript?

JavaScript (JS) es un lenguaje de programación orientado a objetos que se utiliza para añadir funciones complejas a una página web. A diferencia del HTML, que solo organiza contenido, JavaScript es capaz de procesar datos, realizar cálculos y tomar decisiones lógicas. Es un lenguaje de "lado del cliente", lo que significa que el código se descarga en la computadora del usuario y el navegador lo ejecuta al instante sin necesidad de internet constante una vez cargado.

Por ejemplo, JavaScript permite indicar:

  • Cuándo debe mostrarse una ventana de alerta o un aviso.
  • Cómo debe reaccionar un botón al hacer clic sobre él.
  • Qué información debe validarse en un formulario antes de enviarlo.
  • Cómo actualizar parte del contenido de la página sin tener que recargarla entera.

Gracias a esto, el navegador puede ofrecer una experiencia mucho más fluida, convirtiendo documentos estáticos en aplicaciones web interactivas.

2. Integración con HTML

Para que JavaScript funcione, debe conectarse al HTML. Existen dos formas principales:

  • Interna: El código se escribe directamente entre las etiquetas dentro del archivo HTML:
    • <script></script>
  • Externa (Recomendada): Se crea un archivo separado (ejemplo: script.js) y se vincula usando:
    • <script src="js/script.js"></script>

Consejo clave: Se recomienda colocar la etiqueta <script> justo antes de cerrar el </body> .Esto permite que la página cargue visualmente antes de ejecutar la lógica, mejorando la velocidad.

3. La consola del navegador

La consola es una herramienta fundamental incluida en navegadores como Google Chrome. Funciona como una ventana de comunicación directa entre el programador y el navegador, permitiendo ver qué está ocurriendo "detrás de escena" en tu código.

¿Para qué sirve principalmente?

  • Visualizar resultados: Puedes enviar mensajes o valores de tus datos para comprobar que todo funciona bien.
  • Detectar errores: Si algo falla en tu código JavaScript, la consola te mostrará un mensaje en color rojo indicando exactamente en qué línea está el problema.
  • Probar código rápido: Puedes escribir pequeñas instrucciones de JavaScript directamente en ella y ver el resultado al instante.

¿Cómo abrirla?

Para acceder a ella, solo tienes que hacer clic derecho en cualquier parte de tu página web, seleccionar "Inspeccionar" y después hacer clic en la pestaña que dice "Console".

La instrucción estrella: console.log()

Es la herramienta más utilizada para imprimir información. Todo lo que pongas dentro de los paréntesis de un console.log() aparecerá automáticamente en esa ventana gris del navegador, ayudándote a entender el flujo de tu programa.

4. Comentarios en JavaScript

Los comentarios se utilizan para explicar el funcionamiento del código o para dejar notas que no queremos que el navegador ejecute. Son esenciales para mantener el código legible y ayudar a otros programadores (o a ti mismo en el futuro) a entender qué hace cada parte.

Existen dos formas de escribirlos:

  • Comentario de una sola línea: Se utiliza colocando dos barras inclinadas //. Todo lo que escribas después de las barras en esa misma línea será ignorado por el navegador. Se usa para explicaciones rápidas.
  • Comentario multilínea: Se abre con /* y se cierra con */. Es muy útil cuando necesitas escribir párrafos más largos o documentar una función compleja, ya que puede ocupar varias líneas.

Un uso muy común: Además de explicar, los programadores usan los comentarios para "desactivar" temporalmente una línea de código que está dando fallos sin necesidad de borrarla por completo.

5. Resumen

Hemos aprendido que JavaScript es el lenguaje que aporta interactividad a la web, funcionando como el "cerebro" que responde a las acciones del usuario. Vimos cómo integrarlo en el HTML mediante etiquetas o archivos externos, y la importancia de colocarlo al final del documento para optimizar la carga. También exploramos el uso de la consola para detectar errores en tiempo real y cómo los comentarios ayudan a organizar y documentar nuestra lógica de programació
